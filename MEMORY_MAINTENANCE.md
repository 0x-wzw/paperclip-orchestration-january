# MEMORY_MAINTENANCE.md

## Daily Tasks

### Morning Shift (UTC 06:00-12:00)
1. **Weather check** — If Z might be going out today
2. **Calendar review** — Events in next 24-48h
3. **Email scan** — Any urgent unread messages
4. **Memory read** — Today's memory file (memory/YYYY-MM-DD.md)
5. **Heartbeat state** — Check heartbeat-state.json

### Afternoon Shift (UTC 12:00-18:00)
1. **Notifications** — Review any pending notifications
2. **Memory organize** — Read and file recent memory entries
3. **Project check** — Git status on active projects
4. **Agent sessions** — Check on any running sub-agents

### Evening Shift (UTC 18:00-23:00)
1. **Calendar update** — Upcoming events to prepare for
2. **Important matters** — Anything that needs follow-up
3. **Memory pause** — Stop checking memory (save cycles)

## Weekly Review

**Schedule:** Every ~7 days or at end of week

**Actions:**
1. Read through recent memory/YYYY-MM-DD.md files (last 3-7 days)
2. Identify significant events, lessons, or insights worth keeping long-term
3. Update MEMORY.md with curated learnings
4. Remove or archive outdated info from MEMORY.md
5. Clean up heartbeat-state.json (remove stale entries)
6. Archive old daily memory files (>1 month old if possible)

**Review Focus:**
- Decisions made and their outcomes
- Lessons learned
- Patterns observed
- Relationships developments
- Personal growth notes

## Key Principles

1. **Capture what matters** — Don't log everything; log the significant
2. **Update MEMORY.md regularly** — Your long-term memory needs care
3. **Use daily memory for raw logs** — Separate concerns
4. **Be concise** — Memory is limited; quality over quantity
5. **Include context** — Who, what, when, why, how
6. **Tag important items** — Help future search and retrieval

## Check Rotation

Heartbeat checks rotate through categories:
1. Email
2. Calendar
3. Weather
4. Mentions/Notifications
5. Memory
6. Projects

**Rotating every ~30 minutes** ensures comprehensive coverage without repetitive checks.

## What to Capture

### In Daily Memory (YYYY-MM-DD.md):
- Session starts and ends
- Actions taken with context and results
- Decisions made (with brief reasoning)
- Errors encountered and how resolved
- Relevant discoveries
- Human interactions and learnings
- File modifications
- Anything worth remembering for the day

### In MEMORY.md:
- Distilled learnings from daily logs
- Lessons learned over time
- Decisions that shaped long-term direction
- Personas and preferences gathered
- Relationships with key people
- Context that persists across sessions
- Goals and objectives

### In heartbeat-state.json:
- Last check timestamps per category
- Next check scheduled time
- Last heartbeat timestamp
- Status of proactive tasks
